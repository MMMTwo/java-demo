###### 抽象工厂模式
```
        简单工厂模式根据你需要的类型参数,创造出相对应的对象
        例如当前示例:
            用户选购（User）
            销售平台（CarFactory）
            造车公司（Car）
                造小轿车部门（SedanCar）
                造公交车部门（BusCar）
                造自行车部门（BikeCar）
        用户（User）跟销售平台（CarFactory）订车，销售平台（CarFactory）根据用户（User）的需求让汽车制造公司（Car）通知对应的制造部门（*Car） 此处*是（BusCat,BikeCar,SedanCar）其中的某一种,由用户传入的类型变化
-----------------------------------------------------------------------------------------------------------------------------------------
        上面是简单工厂模式的执行步骤
        抽象工厂模式示例:
                    用户选购（User）
                    销售平台（CarFactory）
                        自行车销售部（BikeFactory）
                        公交车销售部（BusFactory）
                        小轿车销售部（SedanFactory）
                    造车公司（Car）
                        造小轿车部门（SedanCar）
                        造公交车部门（BusCar）
                        造自行车部门（BikeCar）
         现在的用户购车流程是:用户（User）提供销售平台（CarFactory）选购汽车，销售平台（CarFactory）将用户所选购的汽车通知对应的销售部（&Factory），由具体的销售部（*Factory）去通知汽车制造部门（*Car）
         此处*是（BusCat,BikeCar,SedanCar）其中的某一种,由用户的选则变化
         此处&是（BusFactory,BikeFactory,SedanFactory）其中的某一种,由用户的选则变化
-----------------------------------------------------------------------------------------------------------------------------------------
比较上述两种方式可见
    简单工厂是通过销售平台（CarFactory）直接联系造车公司（Car）通知造车部门（*Car）
    抽象工厂模式是通过销售平台（CarFactory）联系到具体的销售部（&Factory）然后通过具体的销售部（&Factory）联系造车公司（Car）通知造车部门（*Car）
    
    如果此时造车公司发布新款火车
            造火车部门（trainCar）
            销售火车部（trainFactory）
    那么简单工厂模式需要在销售平台添加选购火车选项这是会改动的之前的代码违反了开闭原则
    如果使用抽象工厂模式只需要添加一个制造方（trainCar）和一个销售部门（trainFactory）不用改动之前的代码符合开闭原则，用户如果选购火车的话直接跟火车销售部门下单即可

```
